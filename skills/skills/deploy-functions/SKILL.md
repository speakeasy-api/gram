# Deploy Gram Functions

Guide for deploying Gram Functions (serverless TypeScript tools) to the Gram platform.

## When to Use

- Deploying a built Gram Functions zip to Gram
- Deploying functions alongside OpenAPI specs in a single deployment
- Setting up CI/CD for function deployments

## Prerequisites

- Gram CLI installed and authenticated (`gram whoami` to verify)
- A built Gram Functions zip file containing compiled code and `manifest.json`
- Target project configured via `--project` flag, `GRAM_PROJECT`, or profile

## Inputs

| Input | Description | Required |
|---|---|---|
| Functions zip | Path or URL to zip file | Yes |
| Slug | URL-friendly identifier (`^[a-z0-9_-]{1,128}$`) | Yes |
| Name | Human-readable display name | No |
| Runtime | Execution runtime (default: `nodejs:22`) | No |

## Outputs

| Output | Description |
|---|---|
| `gram.deploy.json` | Deployment config (stage+push flow) |
| Deployment ID | Returned by `gram push` or `gram upload` |
| Status | `completed`, `processing`, or `failed` |

## Supported Runtimes

| Runtime | Description |
|---|---|
| `nodejs:22` | Node.js 22 (default, currently the only supported runtime) |

## Command: Stage + Push (Recommended)

**Step 1 — Stage the function:**

```bash
gram stage function \
  --slug my-functions \
  --location ./dist.zip \
  --name "My Functions"
```

Runtime defaults to `nodejs:22`. To be explicit:

```bash
gram stage function \
  --slug my-functions \
  --location ./dist.zip \
  --runtime nodejs:22
```

**Step 2 — Push the deployment:**

```bash
gram push --config gram.deploy.json
```

Push with options:

```bash
# Replace all existing artifacts instead of merging
gram push --config gram.deploy.json --method replace

# CI/CD: don't wait for completion
gram push --config gram.deploy.json --skip-poll

# Idempotent deploy (safe to retry)
gram push --config gram.deploy.json --idempotency-key "deploy-$(git rev-parse HEAD)"
```

## Multi-Source Deployments

Stage both OpenAPI specs and functions in the same config, then push once:

```bash
# Stage the API spec
gram stage openapi --slug my-api --location ./openapi.yaml

# Stage the functions
gram stage function --slug my-tools --location ./dist.zip

# Review — both sources appear in gram.deploy.json
cat gram.deploy.json

# Push everything at once
gram push --config gram.deploy.json
```

The resulting `gram.deploy.json`:

```json
{
  "schema_version": "1.0.0",
  "type": "deployment",
  "sources": [
    {
      "type": "openapiv3",
      "location": "./openapi.yaml",
      "name": "my-api",
      "slug": "my-api"
    },
    {
      "type": "function",
      "location": "./dist.zip",
      "name": "my-tools",
      "slug": "my-tools",
      "runtime": "nodejs:22"
    }
  ]
}
```

## Command: Upload (One-Step)

```bash
gram upload \
  --type function \
  --location ./dist.zip \
  --slug my-functions \
  --name "My Functions" \
  --runtime nodejs:22
```

## Example: Build and Deploy

If using the `@gram-ai/functions` SDK with the `gf` CLI:

```bash
# Build the functions (creates dist.zip)
gf build

# Deploy via gf push (handles staging and pushing)
gf push --project my-project
```

Or manually with the Gram CLI:

```bash
# 1. Build your functions
cd my-functions && npm run build

# 2. Verify auth
gram whoami

# 3. Stage and push
gram stage function --slug my-tools --location ./dist.zip
gram push --config gram.deploy.json

# 4. Check status
gram status
```

## Zip File Requirements

The zip file must contain:

- **`manifest.json`** — Generated by `gf build`, describes available tools
- **Compiled JavaScript** — The built function code
- **`node_modules/`** — Dependencies (if any, bundled by the build)

If `manifest.json` is missing, the deployment will fail.

## What NOT to Do

- Do not deploy source TypeScript files — build first with `gf build`
- Do not forget `manifest.json` in the zip — the deployment will fail
- Do not use a runtime other than `nodejs:22` (it's currently the only supported runtime)
- Do not mix up slugs between OpenAPI and function sources — they share a namespace within a deployment

## Troubleshooting

| Problem | Solution |
|---|---|
| "manifest.json not found" | Rebuild with `gf build` — it generates the manifest |
| "unauthorized" error | Run `gram auth` or verify `GRAM_API_KEY` is Provider-scoped |
| Deployment fails | Run `gram status --json` — check error details |
| Wrong project | Run `gram whoami`, use `gram auth switch` to change |
| Zip too large | Check for unnecessary files; `gf build` handles bundling |
